What is Version Control?


Often version control systems are divided into two groups: "centralized" and "distributed".

Version control systems are often divided into two groups: centralized and distributed

Centralized Version Control

Located in 1 place.

People can checkout from different locations and check back in.

 Can be a problem with no access to central repository.
Idea that there is a single copy of your project somewhere that developers commit changes to.

Distributed Version Control

Every developer clones a copy of a respository and has the full history of the project locally.

DVCs Do not rely on a central server to store all versions of project's files. Local has everything the remote repository has since the last sync. Popular DVCs out currently are Mercurial, Git, and Bazaar.

Git

version-control system for tracking changes in computer files.

Git was created by Linus Torvalds in 2005 for development of the Linux kernel.

Git is free and open-source software distributed under the terms of the GNU General Public License version 2.
Primarily used for source-code management, can also be used to track changes of sets of files.
